# mutsubi-dx モデル選択ガイド

**作成日**: 2026-02-16
**目的**: タスクの複雑度に応じた適切なモデル選択でコストを最適化

---

## モデル比較

| モデル | 得意分野 | コスト | 推奨用途 |
|--------|---------|--------|---------|
| **Haiku** | 高速、定型処理 | 低 | 軽量タスク、ログ更新 |
| **Sonnet** | バランス | 中 | 実装、レビュー、大半の作業 |
| **Opus** | 複雑な推論、設計 | 高 | アーキテクチャ、難解な問題 |

---

## タスク別推奨モデル

### Haiku推奨（コスト重視）

| タスク | 理由 |
|--------|------|
| 学び更新（shared-learnings.md） | 定型的な追記作業 |
| 稼働ログ記録 | フォーマット化された記録 |
| 単純なファイル編集 | 置換・追記のみ |
| ステータス確認 | 読み取りのみ |
| テンプレート適用 | パターンマッチ |

**spawn例**:
```javascript
{
  "subagent_type": "general-purpose",
  "model": "haiku",
  "prompt": "shared-learnings.mdに今日の学びを追記してください..."
}
```

### Sonnet推奨（標準）

| タスク | 理由 |
|--------|------|
| コード実装 | バランスの取れた能力 |
| PRDレビュー | 分析と提案が必要 |
| ドキュメント作成 | 構造化と文章力 |
| テスト設計 | 網羅性と論理性 |
| バグ修正 | 原因特定と修正 |

**spawn例**:
```javascript
{
  "subagent_type": "general-purpose",
  "model": "sonnet",  // または省略（デフォルト）
  "prompt": "PRDをレビューして指摘事項をまとめてください..."
}
```

### Opus推奨（品質重視）

| タスク | 理由 |
|--------|------|
| アーキテクチャ設計 | 長期的影響の考慮 |
| 複雑な統合作業 | 多角的な分析 |
| 重要な意思決定支援 | 深い推論 |
| 難解なバグ調査 | 複雑な因果関係 |
| セキュリティレビュー | 細部への注意 |

**spawn例**:
```javascript
{
  "subagent_type": "general-purpose",
  "model": "opus",
  "prompt": "システム全体のアーキテクチャを設計してください..."
}
```

## モデル切り替え方法

### セッション中

```
> /model
```

### spawn時の指定

```javascript
{
  "subagent_type": "general-purpose",
  "model": "haiku",  // "sonnet" | "opus" | "haiku"
  "prompt": "..."
}
```

### デフォルト設定

`/config` でデフォルトモデルを設定可能

## 判断フローチャート

```
タスクを受け取った
    │
    ├── 定型的・単純か？
    │   └── Yes → Haiku
    │
    ├── 複雑な推論が必要か？
    │   └── Yes → Opus
    │
    └── それ以外 → Sonnet
```

---

## 注意事項

1. **Opusの過剰使用を避ける**: 必要な場合のみ使用
2. **Haikuの限界を理解**: 複雑な分析には不向き（下記参照）
3. **チーム全体のバランス**: 全員Opusは非効率
4. **タスク分割**: 大きなタスクを分割し、適切なモデルを割り当て

---

## Haikuの限界と失敗パターン

### Haikuが不向きなタスク

| タスク | 理由 | 推奨モデル |
|--------|------|-----------|
| 複数ドキュメントの比較分析 | 複雑な因果関係の追跡が苦手 | Sonnet |
| PRDの重大問題指摘 | 微細な矛盾検出が不十分 | Sonnet/Opus |
| セキュリティレビュー | 多角的な脅威分析が必要 | Opus |
| アーキテクチャ設計 | 長期的影響の考慮が不足 | Opus |
| 法規制の詳細分析 | ニュアンスの理解が必要 | Sonnet |
| 複雑なバグ調査 | 複合的な原因特定が困難 | Sonnet/Opus |

### Haiku失敗の兆候

以下の兆候が見られたらSonnetに切り替え:

| 兆候 | 対応 |
|------|------|
| 同じミスを繰り返す | → Sonnetで再実行 |
| 指示の一部を無視 | → Sonnetで再実行 |
| 表面的な回答のみ | → Sonnetで深掘り |
| 矛盾した出力 | → Sonnetで検証 |

### Haiku適用可否の判断フロー

```
タスクを受け取った
    │
    ├── 単一ファイルの編集か？
    │   ├── Yes & パターン化された変更 → Haiku OK
    │   └── No or 判断が必要 → Sonnet
    │
    ├── 分析・比較が必要か？
    │   └── Yes → Sonnet/Opus
    │
    └── 定型的な追記か？
        ├── Yes → Haiku OK
        └── No → Sonnet
```
